# Redux and Zustand

## Why State Libraries
They help manage shared and complex state predictably across large applications.

## How Redux Works
1. UI dispatches an action.
2. Reducer computes new state from previous state + action.
3. Store updates.
4. Subscribed UI re-renders.

## Why Redux
- Predictable unidirectional flow.
- Great debugging with DevTools and time-travel.
- Scales well in large teams/projects.

## When to Use Redux
- Complex shared state.
- Multi-step business workflows.
- Need for strict patterns and traceable updates.

## Redux Toolkit (RTK)
- Recommended modern Redux.
- Less boilerplate with `createSlice`.
- Built-in good defaults in `configureStore`.
- Async with `createAsyncThunk` or RTK Query.

## Zustand
- Lightweight store with minimal boilerplate.
- Easy API and quick setup.
- Great for small to medium apps or isolated feature stores.

## Redux vs Zustand (Quick)
- Redux: stronger structure, better for very large apps.
- Zustand: faster development, simpler mental model.
