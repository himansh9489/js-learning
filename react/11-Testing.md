# Testing in React

## Why Testing
- Prevent regressions.
- Improve confidence during refactoring.
- Document expected behavior.

## React Testing Library (RTL)
- Focus on user behavior, not component internals.
- Test rendered output, interactions, and async behavior.

## Unit Testing
- Test small units: utility functions, hooks, isolated components.
- Keep tests deterministic and independent.

## Common Test Cases
- Render correctness.
- Button click and state update.
- API success/loading/error states.
- Conditional rendering and route guards.

## Interview Hack
Say: "I write components in a testable way by separating business logic from presentation and covering key flows with unit tests."
