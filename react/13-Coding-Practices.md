# Reusability, Readability, Modularity, Testability

## Reusability
- Build composable components with clear props.
- Extract repeated logic into custom hooks.

## Readability
- Use meaningful names.
- Keep components small and focused.
- Avoid deeply nested conditionals in JSX.

## Modularity
- Organize by feature/domain.
- Separate UI, business logic, API, and utilities.

## Testability
- Prefer pure functions for logic.
- Keep side effects isolated.
- Use dependency injection where useful.

## Interview Tip
Frame decisions with maintainability: "Will another developer understand and extend this quickly?"
