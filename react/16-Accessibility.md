# Accessibility (A11y) in React

## Why Accessibility Matters
- Inclusive user experience.
- Better usability for everyone.
- Often improves semantic structure and quality.

## Core Practices
- Use semantic HTML (`button`, `label`, `nav`, etc.).
- Ensure keyboard navigation works.
- Manage focus on modals and route changes.
- Use proper labels and ARIA attributes when needed.
- Maintain sufficient color contrast.

## React-Specific Tips
- Prefer native elements over clickable `div`s.
- Handle form validation accessibly.
- Add meaningful `alt` text for images.

## Testing A11y
- Manual keyboard testing.
- Screen reader checks.
- Tooling like `axe` in tests.

## Interview Tip
Mention accessibility from design stage, not as a final checklist item.
